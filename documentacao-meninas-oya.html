Visão Geral do Projeto
O que é o Meninas Oyá e quais problemas resolve

🎯 Propósito

Sistema web para gerenciar professores, alunas, atividades e instituições participantes do programa Meninas Oyá — iniciativa que promove a inclusão de meninas nas ciências e tecnologias.

👥 Perfis de Acesso

ADMIN — Coordenação do programa. Aprova/rejeita professores, gerencia todas as entidades, reseta senhas.
PROFESSOR — Cadastra atividades, gerencia suas alunas, atualiza seu próprio perfil.
Público — Visitantes podem visualizar atividades e fazer pré-cadastro como professor.

📦 Módulos do Sistema

  • Autenticação — Login com JWT + controle de roles
  • Professores — Cadastro com aprovação pendente
  • Alunas — Vinculação por tipo de bolsa/voluntariado
  • Atividades — Publicação com fotos e localização
  • Escolas/Instituições — Catálogo de parceiros
  • Cidades — Geolocalização por UF/município
  • Fotos — Upload de imagens associadas

Segurança — Spring Security + JWT
Configuração do SecurityFilterChain e estratégia de tokens

🔒 SecurityFilterChain

// SecurityConfig.java .csrf(disable()) .cors(withDefaults()) // CorsConfig separado .sessionManagement( STATELESS // sem session server-side ) .authorizeHttpRequests( POST "/auth" → permitAll POST "/api/professores/cadastro" → permitAll GET "/atividades/**" → permitAll GET "/api/cidades/**" → permitAll GET "/api/escolas/simples" → permitAll OPTIONS "/**" → permitAll anyRequest() → authenticated ) .oauth2ResourceServer( jwt → jwtAuthenticationConverter() )

🔑 JWT com RSA (RS256)

Encoder (assinar token)

Usa a chave privada RSA para assinar o JWT via NimbusJwtEncoder. O token contém: issuer, subject (username), issuedAt, expiresAt (+3600s), authorities (SCOPE_*).

Decoder (validar token)

Usa a chave pública RSA via NimbusJwtDecoder.withPublicKey(). O converter extrai o campo authorities e mapeia para SimpleGrantedAuthority.

🔐 Senhas — BCrypt

Todas as senhas são hasheadas com BCryptPasswordEncoder. Na criação do usuário, a senha em texto puro é hasheada antes de persistir. Na autenticação, passwordEncoder.matches() compara.

Frontend — Armazenamento do Token

Após o login bem-sucedido, o Vue decodifica o payload Base64 do JWT (atob(token.split('.')[1])) para extrair o campo authorities.

  • Token salvo em localStorage.setItem("token", ...)
  • Se authorities contém ADMIN → redireciona para /painel-admin
  • Caso contrário → redireciona para /painel

Axios — Interceptor de Requisição

O serviço api.js possui dois interceptors configurados:

  • Request: Lê o token do localStorage e adiciona o header Authorization: Bearer {token} em toda requisição autenticada.
  • Response (erro 401): Remove token, logado e usuario do localStorage — força novo login.

Payload do JWT gerado

{ "iss": "Meninas", "sub": "usuario@email.com", "iat": ..., "exp": ..., "authorities": "SCOPE_PROFESSOR" } — O subject é o username (email normalizado) e authorities usa prefixo SCOPE_ (convertido de ROLE_ na geração).

Controle de Acesso — Roles e Permissões

🔑 Roles do Sistema

SCOPE_ADMIN Admin
  • Criado automaticamente na startup via AdminUserConfig
  • Credenciais via variáveis de ambiente (admin.username, admin.password)
  • Acesso total a todos os endpoints protegidos
  • Pode criar/editar/deletar qualquer entidade
  • Aprova ou rejeita professores pendentes
  • Reseta senha de qualquer professor
SCOPE_PROFESSOR Professor
  • Criado via cadastro público ou pelo admin
  • Acesso restrito às próprias atividades e dados
  • Não pode editar/excluir atividade de outro professor
  • Não pode gerenciar perfil de outro professor
  • Pode publicar atividades, gerenciar suas alunas

Regras de Negócio — Professores
Fluxo de aprovação, validações e restrições de acesso

📋 Fluxo de Cadastro e Aprovação

1. Cadastro Público
POST /api/professores/cadastro
2. Status: PENDENTE
Professor NÃO consegue fazer login
3. Admin Analisa
Painel Admin → Aba "Pendentes"
✅ APROVADO
❌ REJEITADO
APROVADO → pode fazer login e publicar atividades
REJEITADO → login bloqueado com mensagem específica (403)
Admin cria diretamente: POST /api/professores (autenticado como ADMIN) → status já sai como APROVADO, sem passar pela fila de pendentes.

✅ Validações Obrigatórias

CPF Único

Antes de salvar, valida se o CPF já existe. CPF é normalizado: todos os caracteres não-numéricos são removidos (replaceAll("\\D", "")). Deve conter exatamente 11 dígitos.

E-mail Único (como Username)

O e-mail é normalizado para minúsculas (trim().toLowerCase()) e usado diretamente como username no Usuario. Valida duplicidade tanto em Professor quanto em Usuario.

Autorização Granular

Para editar/deletar um professor, o sistema verifica: é ADMIN? Ou é o próprio professor? (professorPodeGerenciar()). Se não, lança 403 FORBIDDEN.

Reset de Senha

Apenas ADMIN pode resetar senha de outro professor via PATCH /api/professores/{id}/reset-senha. A senha é re-hasheada com BCrypt antes de salvar.

Regras de Negócio — Atividades, Alunos e Escolas
Restrições e comportamentos importantes de cada módulo

📚 Atividades

Upload de Fotos

Aceita até 2 imagens por atividade (foto e foto2). Os bytes são armazenados diretamente no banco (bytea no PostgreSQL). O content-type é preservado para servir corretamente via GET /atividades/{id}/foto.

Criação por Professor

O professor logado é associado automaticamente via authentication.getName() (subject do JWT). Não é possível criar atividade sem professor.

Criação pelo Admin

Admin pode criar atividade para qualquer professor. Deve informar o professorId no form-data. Sem professorId, a criação pelo admin retorna 400 BAD_REQUEST.

Edição/Exclusão

Professor só pode editar ou excluir suas próprias atividades. Tentativa de modificar atividade alheia retorna 403 FORBIDDEN. Admin pode tudo.

👧 Alunos

Vínculo por Tipo de Bolsa

Cada aluna tem um tipo de vínculo com o programa:

  • BOLSISTA_ADC — Bolsa ADC
  • BOLSISTA_ICJ — Bolsa ICJ
  • BOLSISTA_IC — Iniciação Científica
  • VOLUNTARIA — Voluntária
Busca por Nome

O endpoint GET /api/alunos?q= filtra por nome contendo o termo (case-insensitive) usando findByNomeContainingIgnoreCase.

Associação a Escola

A escola é opcional no cadastro de aluno. Se informada, o repositório valida que a escola existe (400 se não encontrada). A cidade também é opcional.

🏫 Escolas

Bloqueio de Exclusão

Escola não pode ser excluída se houver alunas vinculadas. O sistema conta com alunoRepository.countByEscola_Id() e retorna 409 CONFLICT com mensagem: "Não é possível excluir: esta escola possui X aluna(s) vinculada(s)."

Contagem de Alunas

O endpoint GET /api/escolas retorna um EscolaResponse (DTO) que inclui o campo totalAlunos calculado em tempo real.

Classificações

Cada escola pode ser classificada por:

  • Tipo: ESCOLA_PUBLICA, ESCOLA_PRIVADA, UNIVERSIDADE, INSTITUTO
  • Nível: EDUCACAO_BASICA, ENSINO_SUPERIOR
  • Natureza: PUBLICA, PRIVADA, COMUNITARIA
  • Esfera: MUNICIPAL, ESTADUAL, FEDERAL, OUTRO

const api = axios.create({ baseURL: "http://localhost:8080" }); // Injeta Bearer token em toda requisição api.interceptors.request.use((config) => { const token = localStorage.getItem("token"); if (token) config.headers.Authorization = `Bearer ${token}`; return config; }); // Em 401: limpa localStorage (força re-login) api.interceptors.response.use( res => res, err => { if (err.response?.status === 401) { localStorage.removeItem("token"); localStorage.removeItem("logado"); } return Promise.reject(err); } );

Configurações de Segurança e Inicialização
CORS, AdminUserConfig, TokenConfig e detalhes de implementação

🌍 CORS — CorsConfig.java

Configuração de Cross-Origin Resource Sharing para permitir requisições do frontend Vue. Definida como bean separado, permite que o Spring Security aplique via .cors(Customizer.withDefaults()).

👑 AdminUserConfig.java

Executa a cada startup (CommandLineRunner):

  • Lê as variáveis ${admin.username} e ${admin.password}
  • Se o usuário admin não existe → cria com role ROLE_ADMIN
  • Se já existe → atualiza username, senha (re-hasheia BCrypt) e role
  • Garante que o admin sempre existe e a senha está atualizada
⚠️ Atenção: A senha do admin deve ser definida via variável de ambiente ou application.properties. Nunca hardcode em código-fonte.

🎟️ TokenConfig.java

  • jwt.issuer — identidade do emissor do token (padrão: "meninas-api")
  • jwt.expires-in — tempo de expiração em segundos (padrão: 3600 = 1h)

🔑 JwtAuthenticationConverter

Converte o campo authorities do JWT em GrantedAuthority:

  • Lê a string "SCOPE_ADMIN" ou "SCOPE_PROFESSOR" do claim
  • Separa por espaço (permite múltiplos roles)
  • Mapeia para SimpleGrantedAuthority
  • O @PreAuthorize("hasAuthority('SCOPE_ADMIN')") usa esse mapeamento

🔐 Detalhes do Gerador de Token

  • Roles são convertidas: ROLE_ADMIN → SCOPE_ADMIN
  • Token válido por 3600 segundos (1 hora)
  • Issuer: "Meninas"
  • Subject: usuario.getUsername() (email normalizado)

Pontos de Atenção e Melhorias Sugeridas
Observações técnicas relevantes para evolução do sistema

⚠️ Pontos de Atenção

Token no localStorage

O JWT é armazenado no localStorage, que é vulnerável a ataques XSS. Em produção, considerar uso de HttpOnly cookies para armazenar o token.

Fotos no Banco de Dados

Imagens são armazenadas como bytea no PostgreSQL. Para volumes maiores, recomenda-se migrar para storage externo (S3, MinIO, etc.).

Sem Refresh Token

O sistema usa apenas access token (1h). Após expirar, o usuário é deslogado (interceptor 401). Implementar refresh token melhoraria a UX.

Sem Paginação

Os endpoints de listagem retornam todos os registros sem paginação. Para escala, adicionar Pageable no Spring Data.

🚀 Melhorias Sugeridas

Notificação por E-mail

Enviar e-mail automático quando professor é aprovado ou rejeitado (Spring Mail / SendGrid).

Recuperação de Senha

Implementar fluxo de "Esqueci minha senha" com token temporário enviado por e-mail.

Paginação e Filtros

Adicionar paginação (Pageable) e filtros compostos nos endpoints de listagem.

Swagger / OpenAPI

Documentação interativa da API com SpringDoc OpenAPI 3 (springdoc-openapi-starter-webmvc-ui).

Testes Automatizados

Adicionar testes unitários nos Services e testes de integração nos Controllers com MockMvc e @SpringBootTest.
